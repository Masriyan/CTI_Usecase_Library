## üîÑ Maintaining Badges

### When to Update Badges:

1. **Use Case Count** - Update when adding or removing use cases
2. **Category Counts** - Update when use cases are added to specific categories
3. **Maintenance Status** - Update if repository becomes inactive
4. **License** - Update if license changes
5. **Documentation Status** - Update as documentation evolves

### Badge Update Checklist:

- [ ] Update count in `docs/badges.md`
- [ ] Update count in README.md badges
- [ ] Update count in README.md footer
- [ ] Update category-specific counts
- [ ] Verify all badge URLs are working
- [ ] Test badge rendering in GitHub preview
